# 基于口令的加密网盘客户端：详细说明文档

## 1. 项目概述
本项目是一个**单机模拟 C/S 架构**的课程设计示例：
- 客户端负责：口令输入、文件选择、加密、解密。
- “服务器端”负责：仅保存密文文件（`server_storage/`）。
- 不涉及真实网络通信、数据库、用户系统。

核心密码学方案：
- **PBKDF2(SHA256)**：从口令派生 32 字节密钥。
- **AES-GCM**：执行认证加密，保证机密性与完整性。

```text
secure_cloud_client/
│
├─ main.py
├─ ui.py
├─ crypto_utils.py
├─ file_manager.py
├─ config.py
├─ logger_utils.py
├─ requirements.txt
│
├─ client_cache/
├─ downloads/
├─ server_storage/
├─ logs/
└─ 项目说明文档.md
```

说明：
- `client_cache/`：保存本地加密输出和下载到本地的密文副本。
- `downloads/`：保存解密后的明文。
- `server_storage/`：模拟服务器，仅存放 `.enc` 密文。
- `logs/system.log`：系统日志。

## 3. 模块职责说明

### 3.1 config.py
- 定义密码学参数常量。
- 定义全部路径常量（`Path` 类型）。
- 提供 `init_directories()` 自动创建运行目录。

### 3.2 logger_utils.py
- 提供 `setup_logger()` 初始化日志。
- 提供 `log_info()`、`log_error()` 统一记录操作日志。
- 日志文件输出到 `logs/system.log`。

### 3.3 crypto_utils.py
- 口令校验：`validate_password()`。
- 密钥派生：`derive_key()`。
- 密文封装/解析：`build_encrypted_blob()`、`parse_encrypted_blob()`。
- 文件加解密：`encrypt_file()`、`decrypt_file()`。

### 3.4 file_manager.py
- 生成唯一文件名：`generate_unique_filename()`。
- 上传密文到服务器：`upload_encrypted_file()`。
- 获取服务器文件列表：`list_server_files()`。
- 从服务器下载密文：`download_encrypted_file()`。

### 3.5 ui.py
- Tkinter 主界面类：`SecureCloudApp`。
- 事件函数：
  - `select_file_event()`
  - `upload_event()`
  - `refresh_server_files_event()`
  - `download_event()`
  - `update_status()`

### 3.6 main.py
- 初始化目录与日志。
- 启动 GUI 主程序。

## 4. 密文封装格式（严格）
二进制结构：
```text
[4字节 magic_header]
[2字节 filename_len]
[filename_len 字节 original_filename(utf-8)]
[16字节 salt]
[12字节 nonce]
[剩余部分 ciphertext]
```

字段含义：
- `magic_header = b"SCF1"`：格式识别。
- `filename_len`：原始文件名长度（无符号 2 字节）。
- `original_filename`：原始文件名，便于解密后恢复名称。
- `salt`：PBKDF2 盐值。
- `nonce`：AES-GCM 随机数。
- `ciphertext`：密文（包含认证标签）。

## 5. 主流程说明

### 5.1 加密上传流程
1. 用户输入口令并选择文件。
2. 客户端校验口令合法性。
3. 读取明文，生成 `salt` 与 `nonce`。
4. PBKDF2 派生密钥。
5. AES-GCM 加密明文。
6. 按协议封装为 `.enc`。
7. 保存到 `client_cache/`（同名自动重命名）。
8. 上传复制到 `server_storage/`（同名自动重命名）。

### 5.2 下载解密流程
1. 用户输入口令并选中服务器密文。
2. 将密文复制到 `client_cache/`（同名自动重命名）。
3. 解析密文 blob，提取 `salt/nonce/ciphertext/filename`。
4. PBKDF2 派生密钥。
5. AES-GCM 解密。
6. 输出到 `downloads/`（同名自动重命名）。

## 6. 返回值规范
业务函数统一返回：
```python
(success: bool, message: str, data: str)
```
例如：
- `(True, "文件上传成功", "test_20260309_142530.txt.enc")`
- `(False, "口令错误或文件已损坏，解密失败", "")`

---

## 7. 异常与提示策略
- GUI 面向用户展示友好中文提示。
- 模块内部记录更详细异常日志。
- 解密失败统一口径：
  - **口令错误或文件已损坏，解密失败**

## 8. 运行方式
在 `secure_cloud_client/` 目录执行：
```bash
python main.py
```

```bash
pip install -r requirements.txt
```
